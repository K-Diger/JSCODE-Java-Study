# Git Flow

## Git flow 란?

Git 을 사용하면서 얻고자 하는 이점은 "분산 버전 관리"일 것이다.

Git을 활용한다면 한 프로젝트 레포지토리에 대해서 브랜치를 나누고, 각 브랜치 별로 별도의 작업을 수행하여 독립적으로 버전을 관리하는 것이 가능해진다.

이때, "브랜치를 나누고" 라는 문장에서 브랜치를 나누는 것에 대한 전략이 있다. 그리고 이것을 Git Flow 라고 한다.

## Git flow 의 종류?

- feature

- develop

- release

- hotfix

- master(main)
